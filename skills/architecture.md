# AI Skill: Application Architecture Context

This document outlines the high-level architecture of the Quiz and Guide platform. Use this context to understand how data flows, how pages are structured, and how core features operate when making future code modifications or adding features.

## 1. Application Routing & Pages

The application is built using React and `react-router-dom` (HashRouter). There are strictly **three core pages**:

* **Home (`/`)**: The main landing hub. It reads the central `manifest.json` file, groups all available content by `category`, and renders a grid of navigable cards.
* **Quiz Page (`/quiz/:slug`)**: The dynamic route for all quizzes. It extracts the `slug`, looks up the corresponding metadata in the manifest, dynamically fetches the specific quiz JSON data, and mounts the `QuizEngine`.
* **Guide Page (`/guide/:slug`)**: The dynamic route for all guides. Similar to the Quiz Page, it fetches the guide-specific JSON data and mounts the `GuideEngine`.

## 2. Data Loading Pipeline

Content is decoupled from the UI to allow endless scalability without modifying React components.

* **The Manifest (`src/data/manifest.json`)**: Acts as the central registry. Every piece of content must be registered here with its `title`, `description`, `slug`, `category`, `type` (quiz/guide), and the `dataFile` name.
* **Dynamic Fetching**: The actual heavy content files (e.g., `js-quiz-basic.json`) live in the `public/data/` directory. The `QuizPage` and `GuidePage` use the browser `fetch()` API to asynchronously load these JSON files only when a user navigates to their specific route.
* **Loading States**: While the `fetch()` is resolving, the pages display a centralized loading spinner.

## 3. UI Display Engines

Once data is fetched, it is passed down to reusable "Engines" wrapped inside a shared `Layout` component.

* **Layout Component (`Layout.jsx`)**: A universal wrapper providing a consistent, mobile-responsive UI. It provides a minimal sticky header (with a Home/Back button and the title) and handles the structural padding/margins for the main content area.
* **Guide Engine (`GuideEngine.jsx`)**: Receives the raw JSON array of guide items. It groups them internally by their specific `category` field and renders them sequentially. It safely parses Markdown via the `marked` library.
* **Quiz Engine (`QuizEngine.jsx`)**: Receives the raw JSON array of quiz questions. It handles complex internal state, including tracking user selections, score calculation, and determining when to reveal answers. 

*Both engines utilize `Prism.js` in a `useEffect` hook to automatically apply syntax highlighting to any code blocks generated by the markdown parser.*

## 4. Core Features & Behaviors

### The Learning Mode Toggle
Available exclusively on the Quiz Page, the Learning Mode significantly alters how a user interacts with the `QuizEngine`.

* **Learning Mode OFF (Assessment Mode)**:
  - The user proceeds through the entire quiz, selecting one option per question.
  - No feedback or validation is provided during the selection process.
  - The user must explicitly click the "Submit Assessment" button at the bottom.
  - Upon submission, the screen jumps to the top, displays the final score, and retroactively reveals all correct/incorrect answers alongside their detailed feedback blocks.

* **Learning Mode ON (Interactive Mode)**:
  - The "Submit Assessment" button is hidden.
  - The moment a user clicks an option on a question, that specific question is instantly "revealed".
  - The UI immediately validates their choice (Green for correct, Red for wrong), highlights the correct answer, grays out unselected options, and drops down the detailed Feedback block.
  - Once a question is answered in Learning Mode, its inputs are disabled to prevent changing the answer.
